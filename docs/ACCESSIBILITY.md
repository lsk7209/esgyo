# 접근성 (A11y) 가이드

## 구현된 접근성 기능

### 1. ARIA 레이블 및 역할

#### 에러 페이지 (`app/error.tsx`)
- `role="alert"`: 에러 메시지 알림
- `aria-live="assertive"`: 즉시 알림
- `aria-atomic="true"`: 전체 메시지 읽기
- `aria-labelledby`: 제목 연결
- `aria-describedby`: 설명 연결
- `role="alertdialog"`: 대화상자 역할

#### 404 페이지 (`app/not-found.tsx`)
- `role="main"`: 메인 콘텐츠 영역
- `aria-labelledby`: 제목 연결

#### 입력 필드
- 모든 입력 필드에 `aria-label` 추가
- `Label` 컴포넌트와 `htmlFor` 연결

#### 버튼
- 모든 버튼에 `aria-label` 추가
- 명확한 액션 설명

### 2. 키보드 네비게이션

#### 키보드 이벤트 처리 (`lib/hooks/useKeyboardNavigation.ts`)
- Enter: 기본 액션 실행
- Escape: 취소/닫기
- Space: 버튼 활성화
- Arrow Keys: 방향 네비게이션

#### Tab 순서
- 논리적인 Tab 순서 유지
- 포커스 가능한 요소에 적절한 `tabIndex` 설정

### 3. 시맨틱 HTML

#### 시맨틱 태그 사용
- `<h1>`, `<h2>`, `<h3>`: 적절한 헤딩 계층
- `<main>`, `<nav>`, `<section>`: 의미 있는 구조
- `<button>`: 클릭 가능한 요소
- `<label>`: 입력 필드 레이블

#### 숨김 콘텐츠
- 장식용 이모지에 `aria-hidden="true"` 적용
- 스크린 리더에서 불필요한 콘텐츠 숨김

### 4. 색상 및 대비

#### 색상 대비
- WCAG AA 기준 준수 (4.5:1 이상)
- 텍스트와 배경의 명확한 대비

#### 색상 의존성 제거
- 색상만으로 정보 전달하지 않음
- 아이콘, 텍스트, 패턴으로 보완

### 5. 포커스 관리

#### 포커스 표시
- 모든 인터랙티브 요소에 포커스 표시
- 키보드 네비게이션 시 명확한 포커스

#### 포커스 트랩
- 모달 및 대화상자에서 포커스 트랩
- Escape 키로 닫기 가능

## 테스트 방법

### 1. 키보드 테스트
1. Tab 키로 모든 인터랙티브 요소 접근 가능한지 확인
2. Enter/Space로 버튼 활성화 가능한지 확인
3. Escape로 모달/대화상자 닫기 가능한지 확인

### 2. 스크린 리더 테스트
1. NVDA (Windows) 또는 VoiceOver (Mac) 사용
2. 모든 콘텐츠가 읽히는지 확인
3. 레이블과 설명이 명확한지 확인

### 3. 자동화 도구
- **WAVE**: 웹 접근성 평가 도구
- **axe DevTools**: 브라우저 확장 프로그램
- **Lighthouse**: 접근성 점수 확인
